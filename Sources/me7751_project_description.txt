ME 7751 Project 2
Design and Analysis of Compliant Mechanisms

Spring 2026

Project Overview
This project focuses on the design and analysis of compliant mechanisms. Students are
expected to select a topic of their own interest and develop a complete analytical, numerical, and
computational study around that topic. The project is intended to connect mechanism design
theory, reduced-order modeling, finite element verification, and modern computational tool development.
The selected mechanism may be planar or spatial, provided that the topic is clearly related to compliant mechanism design and analysis. Students are encouraged to choose a problem
with meaningful engineering context, such as precision positioning systems, force/displacement amplifiers, medical devices, soft robotic structures, deployable systems, or other relevant compliant
mechanism applications.

Project Requirements
Each project must satisfy all of the following requirements.
1. Topic selection. The project topic must be directly related to the design and/or analysis
of a compliant mechanism. The mechanism may be planar or spatial.
2. Analytical modeling. Students must derive an analytical model for the mechanism using
an appropriate reduced-order approach:
• For planar compliant mechanisms, the analytical analysis must be derived using the
pseudo-rigid-body (PRB) model method.
• For spatial compliant mechanisms, the analytical analysis must be derived using the
stiffness matrix method.
The analytical formulation should be clearly presented, with all assumptions, modeling steps,
governing equations, and derived performance metrics documented.
3. Finite element validation. Students must construct finite element analysis (FEA) models
of the mechanism and use the FEA results as the ground truth for validation. The comparison should include appropriate kinematic, stiffness, force, stress, displacement, or other
performance measures relevant to the chosen problem.

Expected Technical Content
The final submission should demonstrate the following elements:
• Clear problem definition and motivation for the chosen compliant mechanism.
• Description of the mechanism configuration, assumptions, materials, geometry, boundary conditions, and intended function.
• Derivation of the analytical model using PRB or stiffness-matrix-based methods, as appropriate.
• Construction and description of the FEA model, including assumptions, element choices when
relevant, and loading/boundary conditions.
• Systematic comparison between analytical and FEA results.
• Discussion of sources of discrepancy, such as modeling assumptions, geometric nonlinearity,
distributed compliance effects, large deformation effects, material idealizations, or numerical
approximations.

• Demonstration and explanation of the interactive GUI app.
• Engineering conclusions and recommendations for improving either the compliant mechanism
design or the reduced-order analytical model.

Suggested Structure for the Report
An effective report will typically include the following sections:
1. Title and abstract
2. Introduction and motivation
3. Mechanism description and problem definition
4. Analytical modeling methodology
5. FEA methodology
6. Results and comparison
7. Interactive app description
8. Discussion
9. Conclusions
10. References

Deliverables
The project submission should include the following:
• The final report formatted as an ASME conference paper using the official ASME template.
• The analytical derivation and supporting equations.
• The FEA model results used for validation.
• The interactive GUI app and any required supporting files or instructions to run it.
• A clear comparison demonstrating the discrepancy between analytical and FEA predictions.

Evaluation Emphasis
Projects will be judged primarily on:
• Relevance and quality of the chosen compliant mechanism problem.
• Correctness and rigor of the analytical derivation.
